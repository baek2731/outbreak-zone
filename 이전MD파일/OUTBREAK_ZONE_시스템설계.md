# OUTBREAK ZONE — 핵심 시스템 설계 문서

> 세계관 · 전사자 시스템 · 치료제 이중구조 · 식별 메카닉
> 이 문서는 "왜 이렇게 설계했는가"를 유기적으로 설명하는 기획 인수인계 문서다.
> 구현 세부(함수명/수치)는 별도 개발현황 문서를 참조.

---

## 0. 한 줄 요약

**무기 없는 회수 요원이, 어떤 좀비가 '구할 수 있는 동료'이고 어떤 좀비가 '죽여 마땅한 크리쳐'인지 식별하며, 한정된 치료제를 자기 생존과 동료 구원 사이에서 분배하는 공포 서바이벌.**

---

## 1. 세계관 (Story Foundation)

### 1-1. 사건의 본질
이 좀비 사태는 단순 전염병이 아니다. **어두운 생명공학 실험**에서 비롯됐다. 실험의 산물인 **악성 개체(크리쳐)**는 인간의 형태를 하고 있지만 인간이 아니며, 역으로 인류를 말살하려는 목적을 가진 적대적 존재다.

### 1-2. 두 종류의 "좀비"
화면상 똑같이 위협적으로 보이지만, 본질이 다른 두 존재가 섞여 있다.

| 구분 | 정체 | 도덕적 위상 | 게임적 처리 |
|------|------|------------|------------|
| **크리쳐** | 생명공학으로 만들어진 인공 개체 | 인류의 적 | 처리 대상 (치료제 무효) |
| **감염자** | 감염된 인간 — 한때 동료였던 존재 | 피해자 | 구원/안식 대상 (치료제 유효) |

플레이어의 가장 근본적인 질문은 **"눈앞의 이것은 진짜 인간이었나, 인간의 탈을 쓴 크리쳐인가"** 이다. 이 질문이 곧 게임의 모든 메카닉을 관통한다.

### 1-3. 최종 목표
건물 가장 깊은 곳의 **발원지(5층)**까지 도달해 근원을 회수한다. 그러나 이 모든 사태를 조율하는 정점 — **지능을 가진 크리쳐(최종 보스)** 가 더 깊은 곳에 존재한다. 5층은 끝이 아니라 더 깊은 심연으로의 입구다.

## 2. 설계 철학 (Why)

### 2-1. "무력함"이 핵심 재미다
이 게임은 좀비를 사냥하는 액션이 아니다. **좀비를 못 이기는 무력한 요원**의 공포가 정체성이다. 따라서:

- 플레이어는 **무기를 들지 않는다.**
- 좀비를 "죽이는" 것이 아니라 전투는 **"뿌리치고 도망"** 이다.
- 유일하게 능동적으로 좀비에게 영향을 주는 수단은 **치료제(구원)** 뿐이다 — 폭력이 아니라 구원.

> 이 원칙을 어기는 순간(무쌍 액션이 되는 순간) 공포는 사라진다. 모든 설계는 이 무력함을 지키는 방향으로 수렴해야 한다.

### 2-2. "식별"이 모든 긴장의 근원이다
지뢰찾기 컨셉의 본질은 **정보 부족 상태에서 위험을 식별해 나가는 것**이다. 크리쳐/감염자 구분은 이 컨셉의 완성형이다.

- 기본 소나: 빨간 핑 — 병원체인지 크리쳐인지 감염자인지 **구분 불가** (공포)
- 정밀 소나: 그 존재의 **정체를 판별** — 죽일 것인가 구할 것인가

식별이 곧 생존이고, 식별이 곧 자원 절약이며, 식별이 곧 도덕적 판단이다.

### 2-3. 모든 시스템은 하나로 맞물려야 한다
따로 노는 기능을 더하지 않는다. **치료제 하나가 생존·구원·청소를 동시에 수행**하고, **정밀 소나가 그 모든 판단의 정보원**이 되며, **전사자 풀이 플레이어의 과거를 현재의 위협으로 되돌린다.** 시스템들이 서로를 강화한다.

## 3. 핵심 메카닉: 치료제 이중구조

### 3-1. 치료제란
한정된 소모성 자원. 산소 캡슐과 **명확히 구별되는 별개 자원**이다.

| 자원 | 역할 |
|------|------|
| 산소 캡슐 | 산소(생존 타이머) 회복 |
| **치료제** | 감염도 회복(자가) / 좀비에게 사용(구원) |

### 3-2. 치료제의 세 갈래 선택
치료제를 1개 주울 때마다 플레이어는 갈등한다.

```
치료제 1개를 어디에 쓸 것인가?
├── 내 감염도 회복         → 생존 우선
├── 감염자(동료) 안식      → 구원 + 전사자 풀 청소
└── 아낀다                → 다음을 대비
```

이 딜레마가 로그라이크 자원 관리의 핵심이다. **크리쳐에게 쓰면 완전 낭비**이므로, 식별(정밀 소나)이 낭비를 막는 필수 행위가 된다.

### 3-3. 치료 = 안식 (톤 결정)
감염이 이미 진행된 감염자는 인간으로 되돌릴 수 없다. 치료제는 **고통 없이 보내주는** 역할 — 슬프지만 존엄한 죽음이다.

> "구할 수 있을 줄 알았는데, 할 수 있는 건 편히 보내주는 것뿐이었다."

이 정서가 감염자 안식의 핵심이다. 전사자 풀에서 제거되는 것도 "안식"으로 자연스럽게 설명된다.

- (확장 여지) 감염 진행도에 따라 초기=치료(구원) / 말기=안식(죽음) 분기 가능. "빨리 발견할수록 살릴 수 있다"는 추가 긴장. 단 스코프 확대 요소 — 후순위.

## 4. 전투와 선택의 순간

### 4-1. 통합된 전투 흐름
감염자든 크리쳐든 접촉하면 **동일하게 전투(F연타)가 발생**한다. 겉보기로는 구분되지 않기 때문이다.

```
좀비 접촉 → 전투 발생 → F키 연타
                  ↓
          게이지 50% 도달
                  ↓
      [선택지 등장] 치료제 사용 가능
        ├── 사용함
        │     ├── 감염자였다 → 안식 성공 (전투 즉시 종료, 산소 절약, 전사자 풀 청소)
        │     └── 크리쳐였다 → 효과 없음 (치료제 낭비 + 계속 연타해야 함)
        └── 안 씀
              └── 게이지 100% → 기존처럼 뿌리치고 도망 (산소 소모)
```

### 4-2. 왜 50% 지점인가
- 너무 낮으면: 판단할 새가 없다
- 너무 높으면: 거의 도망 직전이라 선택 의미가 약하다
- 50%: 위험한 순간에 자원 사용을 결정 — 긴장 유지. (플레이테스트로 조정)

### 4-3. 식별 여부에 따른 선택지 차이
정밀 소나로 미리 식별해 둔 좀비라면 확신을 가지고, 아니면 도박이 된다.

```
[식별된 감염자와 전투] → "치료제 사용 (감염자 확인됨)" → 확신
[미식별 좀비와 전투]   → "치료제 사용 (정체 불명)"   → 도박
```

이로써 정밀 소나의 가치가 전투에서 직접 체감된다. **미리 식별 vs 현장 도박**의 선택이 생긴다.

### 4-4. 식별 실패의 대가 (자연 발생)
별도의 인위적 페널티가 필요 없다. **크리쳐에게 치료제를 쓰면 그냥 효과 없음 = 귀한 자원 낭비 + 전투 시간 손해.** 자원 손실 자체가 페널티다.

## 5. 전사자 시스템 (메타 진행의 기둥)

### 5-1. 핵심 개념
플레이어가 죽으면 전사자로 기록된다. 그중 **감염사한 요원은 다음 런에서 감염자(좀비)로 등장**한다. 즉 **플레이어 자신의 죽음이 자신의 미래를 위협한다.**

### 5-2. 왜 이게 우아한가 — 반복사망 패널티
로그라이크의 고질병은 "저층에서 일부러 죽어 안전하게 DNA만 파밍"하는 악용이다. 보통 저층 보상을 깎아 막지만 초보자도 손해 본다.

이 시스템은 다르다.
```
저층에서 죽는다
→ 그 층 전사자 풀에 쌓인다
→ 다음 런에 그 요원이 감염자로 등장
→ 그 층이 점점 위험해진다
```
패널티가 외부 강제가 아니라 **서사적으로 자연 발생**한다. "내가 버린 동료들이 나를 막는다"는 죄책감 서사까지 따라온다.

### 5-3. 사망 유형과 변이의 연결
"어떻게 죽었는가"가 "무엇이 되는가"를 결정한다. (오염으로 특수 좀비가 생기는 이유를 여기서 설명)

| 사망 유형 | 발생 경로 | 변이 결과 | 전사자 풀 |
|----------|----------|----------|----------|
| 감염사 | 감염 100% 도달 (산소 있음) | 표준 감염자 | 등록 O |
| 질식변이 | 산소 0 → 오염 공기 노출 → 감염 | 질식형 (SENSOR 계열) | 등록 O |
| 피습사 | 전투 패배로 산소/감염 폭증 | 추격형 (STALKER 계열) | 등록 O |
| 붕괴사 | 폐허 위험 구역 (소나로 탐지 가능) | 육체 손상 — 변이 안 됨 | 등록 X |

> **크리쳐는 인간이 아니므로 전사자 풀과 무관하다.** 전사자 풀은 오직 감염사한 전 플레이어(요원)만 등록된다.

> 현재 코드의 사망은 2종(`death`/`infected`)이며 감염사/질식사가 사실상 겹쳐 있다. 사망 유형을 늘리려면 **죽는 경로 자체를 분기**해야 한다 (사망 시점의 oxygen/infection 값 + 최근 전투 결과로 판정).

### 5-4. 악순환 방지 — 반드시 필요
패널티가 실력 부족을 가속하면 신규 플레이어가 영구히 막힌다(1층에서 죽음→1층 더 어려워짐→더 죽음). 다음 장치로 막는다.

1. **전사자 풀 상한** — 층당 최대 N명
2. **시간 감쇠** — 오래된 전사자는 풀에서 빠짐 (성불)
3. **치료제 안식 = 청소** — 감염자(전사자)를 치료제로 안식시키면 풀에서 제거 + DNA 보너스. **패널티를 플레이어가 직접 청소 가능**
4. **첫 N런 면제** — 튜토리얼 직후 몇 런은 전사자 미적용

> 3번이 가장 우아하다: 패널티이자 보상 서사이며, 악순환을 플레이어가 끊을 수 있다. 치료제 이중구조와 자연스럽게 연결된다.

### 5-5. 월드 변질 (강화와 위험의 동시 진행)
누적 전사자/런 수가 임계치를 넘으면 새로운 변이체가 해금된다.
```
누적 전사자 임계치 돌파 → 새 변이체 등장 (월드 상태 변화)
```
플레이어가 강해지는 만큼(메타 강화) 세계도 위험해진다(메타 변질). 두 축이 동시에 굴러 긴장이 유지된다.

## 6. 식별의 이스터에그 층위

전사자 시스템은 **전면 기능**이지만, 디테일은 **은밀하게** 둔다.

- **공개되는 룰**: "감염자는 좀비가 되어 돌아온다 / 치료제로 안식시킬 수 있다" — 튜토리얼에서 명시
- **은밀한 디테일**: 그 좀비가 **구체적으로 누구인지**(UNIT-NN, 사망 층/원인 같은 인식표)는 **정밀 소나로 들여다봐야** 보인다

```
일반 시야      → 그냥 감염자 좀비
정밀 소나 조준 → 인식표 표시: "UNIT-07 · 1층 · 감염사"
                → 플레이어: "어? 저거 내가 죽은 요원이잖아"
```

기능은 전면에, 발견의 정서는 은밀하게 — 두 층으로 나눈다.

## 7. 정밀 소나의 격상

정밀 소나는 더 이상 단순 "구분 표시기"가 아니다. **게임의 모든 핵심 판단의 정보원**이다.

```
정밀 소나 1회 사용으로 얻는 정보:
├── 이것은 크리쳐인가 감염자인가 (치료제 낭비 방지)
├── 감염자라면 누구인가 (전사자 인식표 — 이스터에그)
└── 전투 진입 시 확신 vs 도박 결정
```

정밀 소나가 시작 시 2개 제한이라는 점이 이제 매우 무거운 선택이 된다. "이 순간 식별에 쓸 것인가, 아껴둘 것인가."

## 8. 튜토리얼의 역할 (UNIT-00)

튜토리얼은 단순 조작 학습이 아니라 **세계관의 핵심 규칙을 체득**시키는 장치다.

```
튜토리얼 (UNIT-00, 무조건 사망)
1. WASD / 기본 소나 조작
2. 정밀 소나로 크리쳐 vs 감염자 식별 시연
3. "감염자는 동료였다" — 치료제로 안식시키는 의미 학습
4. "크리쳐는 적이다 — 치료제가 통하지 않는다" 학습
5. 다수의 크리쳐에게 둘러싸여 사망 → UNIT-00 전사 기록 등재
   → 이 UNIT-00이 다음 런에서 첫 감염자(첫 안식 대상)로 등장
```

**플레이어의 첫 죽음(UNIT-00)이 곧 첫 안식 대상이 된다.** 튜토리얼이 메타 시스템의 시작점이 되는 강력한 연결.

(설계 연출 아이디어: 어두운 일자 복도 탐색 → 소나 → 빨강 점 하나 → 회수 → 다시 소나 → 빨강 점 가득 → 좀비 들이닥침 → 암전 → UNIT-00 전사)

### 8-1. 실제 구현 현황 (설계 대비 확인된 차이) — 갱신

튜토리얼은 이제 전체 시퀀스가 완성되어 위 설계 의도를 거의 그대로 따른다 — 오히려 더 정교해졌다:
- 기본 소나(오렌지/옐로우 위험도 시연) → 회수 2회 → **평시 치료제 자율선택**(설계엔 없던 디테일, 자가치료 D키 학습 겸용) → 소나 재사용 → 3번째 회수 시점에 좀비 습격 → 전투(F연타, 80% 시점 강제선택) → 정밀소나로 식별("미확인 크리쳐 접근...정밀소나 사용") → 좀비떼 노출 → 암전.
- "치료제가 크리쳐엔 안 통한다"는 학습(설계 4번 항목)은, 습격 좀비가 식별 전 상태로만 다뤄지고 전투 자체가 항상 같은 결말(좀비 제거)로 수렴하도록 만들어져 있어, **"치료제 무효"를 직접 체감시키는 장면은 아직 구현되어 있지 않다.** 설계 의도와 비교하면 빠진 디테일.

> **⚠️ 확인된 설계-구현 갭(중요)**: 설계는 "UNIT-00 사망 → 전사자 풀 등재 → 다음 런의 첫 안식 대상"을 핵심 연결로 명시했으나, **실제 구현된 튜토리얼 완주 경로(`goToLobbyFromTutorial()` → `incrementUnit()` → `showTitle()`)에는 전사자 풀 등록(`addToFallenPool`) 호출이 없다.** 현재 `ensureUnit00Fallen()`(UNIT-00을 전사자 풀에 강제 등록하는 함수)은 **DEV 패널의 "기록 초기화" 버튼**(튜토리얼을 건너뛰고 바로 테스트하는 용도)에서만 호출되고 있어, 실제로 튜토리얼을 끝까지 플레이해 완료한 플레이어에게는 이 연결이 발생하지 않는다. 설계 의도를 살리려면 `goToLobbyFromTutorial()` 안에서 `addToFallenPool()` 호출을 추가하는 보강이 필요 — **다음 작업 후보로 남김.**

## 9. 엔딩과 떡밥

### 9-1. 5층 근원 회수
발원지 가장 깊은 곳의 근원 병원체를 회수하면 ALL CLEAR.

### 9-2. 엔딩 연출 (다크 톤)
```
근원 회수 → 화면 암전 → "발원지 봉쇄 완료"
→ 짧은 정적 → 더 깊은 곳에서 무언가 꿈틀거리는 암시
→ "...아직 끝나지 않았다" (지능형 크리쳐의 존재 암시)
→ ALL CLEAR
```

### 9-3. 엑스트라 스테이지 떡밥
5층 이후, 지능을 가진 크리쳐(최종 보스)와 더 강한 변이체(그랩형/오염형 등)가 기다린다. 떡밥에서 이들의 존재를 암시한다.

### 9-4. 감염/사망 엔딩 연출 (미완성 — 필수 작업)
- 감염사(좀비화): 사망과 구별되는 연출 (보라 톤, 변형 연출)
- ALL CLEAR: 5층 탈출 시 별도 연출

## 11. 미결정 사항 (플레이테스트/추후 결정)

| 항목 | 내용 |
|------|------|
| 치료제 입수 경로 | 맵 드랍? 희소성? (너무 흔하면 딜레마 소멸) |
| 치료제 효과량 | 자가 감염 회복량 / 감염자 안식에 필요한 개수 |
| 50% 수치 | 전투 중 선택지 등장 시점 — 플레이테스트 |
| 크리쳐 낭비 후 처리 | 치료제 무효 시 게이지 리셋 등 추가 페널티 여부 |
| 사망 유형 분기 | 2종 → 4종 확장 시점 (지금? 엑스트라?) |
| 크리쳐/감염자 비율 | 층당 비율 — 회수 위주 vs 전투 위주 밸런스 |
| 전사자 풀 상한 N | 층당 최대 전사자 수 |
| 첫 면제 런 수 | 신규 보호 기간 |
