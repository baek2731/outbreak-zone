# OUTBREAK ZONE — 개발 현황 인수인계 문서

> 브라우저 기반 로그라이크. 탑다운 뷰, 지뢰찾기 컨셉.
> 소나로 병원체/좀비 탐지 → 병원체 회수 미니게임 → 좀비 전투 → 층별 탈출.
> 크레이지게임즈 출품 목표. 그래픽은 추후 도트(픽셀) 스타일로 교체 예정.

## 2. 게임 상태 머신 (GAME_STATE)

```
'TITLE' → 'INTRO' → 'PLAYING' → 'ESCAPED' / 'GAMEOVER'
```

| 상태 | 설명 | 입력 |
|------|------|------|
| TITLE | 타이틀 화면 | 스페이스/엔터 → 기지 |
| INTRO | 스테이지 인트로 | 스페이스/엔터 → 시작 |
| PLAYING | 게임 플레이 | 전체 게임 키 |
| ESCAPED | 탈출/출구 도달 | 버튼만 |
| GAMEOVER | 사망/좀비화 | 버튼만 |

**중요**: `update(dt)` 함수는 `GAME_STATE !== 'PLAYING'`이면 즉시 리턴.
출구 닿는 순간 ESCAPED로 전환되어 좀비/산소/감염 전부 정지.

## 3. 전체 게임 흐름

```
타이틀 → (기지로 이동) → 기지(허브)
  ├── 작전 개시 → UNIT-N 부여 → 인트로 → 게임
  ├── 스테이터스/특성/유물 강화
  └── 전사자 기록 확인

게임 플레이
  ├── 사망/좀비화 → GAMEOVER → 기지로 복귀
  └── 출구 도달 → ESCAPED
        ├── 전부 회수 → 복귀 or 다음 스테이지
        └── 미회수 → 조기복귀 or 재탐사(4초 쿨타임)
```

타이틀은 입구만. 기지가 메인 허브. 게임 종료 후 항상 기지로 복귀.

## 4. 핵심 시스템

### 4-1. 소나
- 기본 소나(F): 병원체+좀비 통합 탐지(구분 불가)
- 정밀 소나(G): 병원체 X / 좀비 눈 아이콘 구분, 시작 시 2개 지급
- 차징식 (F/G 누르고 있다가 떼면 발동), 반경은 차징 시간 비례
- 소나 발동 = 소음 발생 → 파동이 좀비에게 닿으면 반응
- 소나 발동 시 현재 서 있는 병원체 자동 탐지

### 4-2. 병원체 회수
- 소나 탐지(`MAP.detected`) 필수 → E키 회수
- WASD 방향키 패턴 미니게임 (스테이지별 3~5자)
- 실패 시: 감염 증가 + 주황 소음 파동
- 성공 시: `checkPatrolPhase()`, `updateGuardHomePoints()` 호출

### 4-3. 전투
- F키 연타 게이지 힘싸움 (좀비가 지속 drain)
- drain = 좀비 상태(CHASE/SEARCH/WANDER) × 타입 보정
- 전투 후 무적 3초(`MG.postCooldown`)
- 출구 타일 위 / ESCAPED·GAMEOVER 상태에선 전투 차단

### 4-4. 산소/감염 (HP 대체)
- 산소(0~100) + 감염(0~100)
- 스테이지별 자연 감소: `drainPerStage:[2,3.5,5.5,7,9]` × `_drainMult`
- 산소 60% 이하 → 감염 시작 / 산소 0 → 감염 가속
- 층 이동: 산소 +30%(`stageHeal`) 보충, 감염 누적 유지
- 캡슐: 회복량 +33%(`capsuleHeal`)

## 5. 좀비 AI

### 상태 머신
```
WANDER → (소음/시야) → SEARCH/CHASE → (기억 소멸) → WANDER
```
- BFS 경로 탐색(`zombieNextStepDir`) — 벽 우회 추격
- 소음 반응: `triggerNoise(x, y, radiusTiles, sourceType)` — 'sonar' / 'noise' 구분
- 새 소음 = 무조건 목표 갱신 + memoryTimer 리셋 (거리 비교 없음)

### 특수 좀비 (config.zombieTypes)
| 타입 | 색상 | 특징 |
|------|------|------|
| BASIC | #553333 | 기본 |
| SENSOR | #ffcc00 | sensorRange 3 (소나 파동 밖에서도 감지) |
| GUARD | #0088ff | 시야각 110°, 병원체 2~5타일 순찰 |
| STALKER | #8800ff | chaseMemory 8초, 빠름 |
| RUSHER | #ff6600 | CHASE 시 rushSpeed 0.85 |

### 층별 구성 (config.zombieComposition)
| 층 | 구성 |
|----|------|
| 1 | BASIC×1 |
| 2 | BASIC×2 |
| 3 | BASIC×2 + SENSOR×1 |
| 4 | BASIC×2 + SENSOR×1 + GUARD×1 |
| 5 | BASIC×1 + SENSOR×1 + STALKER×1 + RUSHER×1 + GUARD×1 |

### 가드 순찰 (zombieGuardPatrolVec)
- 홈 = 가장 가까운 병원체
- 2타일 미만 → 밀려남 / 2~5타일 → 배회 / 5타일 초과 → BFS 복귀
- 병원체 회수 시 홈 자동 갱신

### 패트롤 강화 (checkPatrolPhase) — 고정 개수 기반
`config.stages[].patrolThresholds:[1단계,2단계,3단계]`
- 1단계: 좀비 `extraSpawn`체 추가 스폰(SEARCH)
- 2단계: `patrol.speedMult = 1.2`
- 3단계: `patrol.speedMult = 1.4`, `patrol.fovMult = 120/90`

## 7. 기지 / 메타 시스템

### DNA 기록 (localStorage)
| 키 | 내용 |
|----|------|
| `outbreak_records` | 런별 상세 배열 (unit, stage, exitType, collected, infection, elapsed) |
| `outbreak_total_dna` | 누적 DNA 총량 |
| `outbreak_upgrades` | `{id: level}` 강화 레벨 |
| `outbreak_unit_number` | 현재 요원 번호 (UNIT-N) |

### 복귀 방식별 DNA 획득
- 일반/조기탈출 = 회수량 전부
- 사망/좀비화 = `Math.floor(회수량 / 2)`
- 저장 시점: `showGameOver()` / `showEscaped()` (중복 방지 `recordSaved` 플래그)

### 스테이터스 6종 (LOBBY.status) — 기본 능력치
방독면 내구도 / 산소 필터 / 항바이러스 / 체력 단련 / 방호복 강화 / 산소 캡슐 개량

### 특성 4종 (LOBBY.trait) — 오브젝트 영향
정밀 해체(패턴-1) / 소음 억제(소음반경-1) / 소나 증폭(반경+1) / 감염 저항(회수 성공 시 감염 0)

- `applyUpgradeEffects()` — 작전 개시 시 CONFIG에 반영
- `getUpgradeEffects()` — 런타임 참조 (패턴 길이, 소음 반경, 감염 저항)
- 강화 초기화 버튼 — DNA 전액 환불

### 기지 UI
- 사이드바(작전개시/탭/초기화/타이틀) + 콘텐츠
- 탭: 스테이터스 / 특성 / 유물(🔒) / 전사자 기록
- UNIT-N 요원 번호, 폰트 크기 `--lb-base` CSS 변수(기본 13px)
- DEV에서 폰트 1px 단위 조절 가능

## 8. 미완성 / 남은 작업

### 🔴 필수 (출품 전)
- **튜토리얼** — 첫 플레이어 무조건 사망, UNIT-00으로 전사자 기록 등재. 기본 조작만 학습
- **유물 시스템** — 해금 조건 충족 후 구매 (특성과 달리 메카닉 변화). 현재 탭 잠금 상태
- **사운드** — 소나/회수/전투/경고 효과음
- **픽셀 디자인** — 도트 스타일, UI 전면 교체 예정 (현재 데이터/로직 중심 설계)
- **모바일 터치 대응** — 현재 키보드 전용
- **출품 빌드** — DEV 패널/로그 숨김

### 🟡 미완성 코어
- 감염(좀비화) 엔딩 연출
- ALL CLEAR 엔딩 연출
- 전사자 기록에 튜토리얼 UNIT-00 첫 줄 등재

### 🔵 밸런스 (플레이테스트 필요)
- DNA 소모 수치 (스테이터스 5~20, 특성 8~25)
- 업그레이드 효과 수치
- 패트롤 강화 타이밍
- 전투 게이지 drain 속도

### 추천 순서
```
감염/클리어 엔딩 → 튜토리얼 → 유물 → 사운드 → 픽셀 → 모바일 → 출품
```

## 9. 주요 설계 결정 (DECISION)

- 병원체 = DNA 수거 대상 (제거 아님). 죽으면 절반만 회수
- 이동 소음 없음 — 소나/회수실패/병원체밟기만 소음 발생
- 특수 좀비는 5층까지 배치. 더 강한 것(그랩형, 오염형)은 엑스트라 스테이지용 (5층 완결 후)
- 튜토리얼 요원은 무조건 사망 → UNIT-00 전사자 기록 첫 줄
- 유물은 해금 조건 + 구매 구조 (특성과 다름)
- UI는 나중에 도트로 전면 교체 — 현재는 데이터/로직 중심
- 기지가 메인 허브. 타이틀은 입구만
- localStorage는 조작 가능 — 싱글 로그라이크라 크게 문제 안 됨. 출품 시 크레이지게임즈 SDK 세이브로 교체 고려
