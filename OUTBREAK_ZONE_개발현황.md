## 1. 프로젝트 개요

**장르:** 브라우저 탑다운 3/4뷰(엔터더건전 시점) 로그라이크 서바이벌
**컨셉:** 지뢰찾기 메카닉 기반 병원체 회수 + 좀비 공포
**출품 목표:** CrazyGames. **모바일은 가로 고정**으로 확정.

**로컬 실행은 반드시 `python -m http.server`** (file://는 CORS로 사운드 fetch 404남).

### 핵심 게임플레이 루프
```
기지 출발 (Space)
→ 소나(F)로 병원체/좀비 탐지
→ 병원체 회수 미니게임 (E + 방향키 패턴)
→ 좀비 조우 시 전투 미니게임 (F 연타)
  → 80% 게이지 시 치료제 선택지 (Y/N)
→ 전체 회수 후 출구 탈출 or 기지 복귀
→ DNA로 기지 업그레이드 → 다음 런
```

## 2. 세계관

- 상부조직(표면=병원체회수 / 실제=ORIGIN 제어권 탈환·요원 소모) → **ORIGIN**(이 건물 실험체, 지성 가진 여성형, 인간 증오·지배욕, 크리쳐 생성) → 요원(플레이어, 진실 모른 채 소모).
- 플레이어는 무기 없이 회수 도구 + 치료제만 보유.

### 두 종류의 적 (정밀소나 G로 식별)
| 구분 | 정체 | 전투 승리 | 치료제(Y) |
|------|------|----------|----------|
| **크리쳐** `[C]`빨강 | ORIGIN 피조물 | 워프로 이탈(먼 곳 순간이동) | 무효 + 새 크리쳐 강제 스폰 |
| **감염자** `[I]`보라 | 전 동료(전사자) | 소멸 + 크리쳐 대체 스폰 | 안식 + DNA 보너스 + 전사자 풀 제거 |

### 치료제 이중구조
생존(D 자가치료) / 구원(감염자 안식) / 비축. **엔딩 분기**: ORIGIN 결전 시 치료제 투입(구원) / 미투입(섬멸).

### 엔딩
- **감염사:** 보라 오버레이 페이드인 → ☣ 좀비 전환 패널
- **ALL CLEAR (5층):** 터미널 타이핑(ORIGIN 기밀 문서) → 빨간 눈 → TO BE CONTINUED. Space/클릭으로 10배속 빨리감기.

## 3. 키 매핑 (점검 완료, 충돌 없음)

| 컨텍스트 | 키 | 기능 |
|---------|----|------|
| 전역 | ESC | 일시정지 토글 (PLAYING↔PAUSED) |
| TITLE | Space/Enter | 로비로 |
| LOBBY | Space | 작전 개시 |
| INTRO | Space/Enter | 인트로 닫기 |
| PLAYING | 방향키 | 이동 / 미니게임 패턴 |
| PLAYING | F | 소나(꾹 차징→뗌 발사) / 전투 연타 / 100% 승리 |
| PLAYING | G | 정밀 소나(진영·인식표 식별) |
| PLAYING | E | 병원체 회수 미니게임 |
| PLAYING | D | 치료제 자가 사용(감염 -10%) |
| 전투 80%선택지 | Y / N | 치료제 투여 / 계속 싸우기 |
| **출구팝업(ESCAPED)** | **Enter** | **기지 복귀** |
| **출구팝업(ESCAPED)** | **Space** | **진행(다음층/재탐사)** — Enter와 상호배타 |
| **GAMEOVER** | **Space/Enter** | **기지 복귀** |

**튜토리얼 전용 분기** (TUT_ACTIVE일 때만 다르게 동작 — 본게임 키맵과 동일한 키를 재사용하되 가드로 단계 제한):
- F: `sonar_prompt`/`sonar_prompt_2` 단계에서만 소나 차징 허용. 충전시간은 항상 최대로 강제(짧게 눌러도 최대반경).
- G: `precise_prompt` 단계에서만 정밀소나 허용(그 외 튜토리얼 단계는 비활성).
- Y/N: `serum_prompt` 단계(평시 자율선택)에서 별도 분기. 전투 중 Y/N(강제선택)과는 다른 코드 경로.
- D: `serum_use_wait` 단계(Y선택 후 실제 사용 확정)에서만 허용. 그 외 튜토리얼 단계는 비활성.

**모바일(가로):** 조이스틱(이동) + 액션버튼 G/D/E/F(2열, F 최대) + 일시정지. 전투 시 Y/N 자동 전환.
**⚠️ 미확인**: 튜토리얼 신규 분기(평시 Y/N, D키, G키)가 모바일 터치 버튼과 연결되는지 미검증 — 아래 6-1 참조.

## 4. 구현 완료 시스템 ✅

소나(기본/정밀, 식별 유지) · 병원체 회수 미니게임 · 전투 미니게임(F연타, 80% 선택지, e.repeat 꾹누름 방지) · 치료제(자가/안식/무효) · 전사자 시스템(풀 등록·안식 시 풀 청소) · 패트롤 3단계 · 산소/감염 · 기지 업그레이드(스테이터스6+특성4, 감염저항 Lv1 30%/Lv2 50%) · 사운드(SFX25+BGM7, zombie_chase 1~3 랜덤) · 플로팅 팝업+공지 · 엔딩 연출 · "그는 사람이었어" 연출 · **튜토리얼(UNIT-00) 전체 — 신규 완성**.

### 이번 세션(들)에서 완료한 주요 작업 — 튜토리얼 전체 구현

처음엔 빈 상태에서 시작해 약 15차례의 반복 수정을 거쳐 완성. 흐름 자체와 디테일이 여러 번 바뀌었으므로, **"왜 이렇게 됐는가"의 맥락이 중요**:

#### 4-1. 전체 흐름 (최종본)
```
world_intro(풀스크린 암전, Space)
→ 도착보고(Space) → 산소안내(Space, "생존시간 약50초") → 이동지시(자동진행)
→ moving → sonar_prompt(F차징, 자동진행) → sonar_result(오렌지+옐로우 언급)
→ mine_collect(1번째 A) → mine_collect_2(2번째 B) → [결과표시 1초]
→ serum_prompt(Y/N, 플레이어 위 UI, 5초타임아웃→N)
   ├ Y→serum_use_wait(D키 안내)→D입력→실제 치료제 적용(오염도 감소)
   └ N→"이건 내가 쓸 게 아니야. 그들을 위해 남겨둬야 해."
→ sonar_prompt_2(소나 재사용 안내) → sonar_result_2 → mine_collect_3(3번째 C=습격 트리거)
→ ambush(전투: "코드레드!"→player.serum 유무로 대사 분기)
→ [전투: F연타 10번 안팎, 80%서 Y/N선택지(7초), 결과 무관 페널티없이 크리쳐 제거로 수렴]
→ aftermath(화이트플래시→정적→비네팅 강한 수렴)
→ precise_prompt("미확인 크리쳐 접근...정밀소나 사용"+[G]안내)
→ [G입력]→precise_revealed(좀비 6마리 노출, 비네팅을 소나 핑 확산 비율에 직접 연동해 해제)
→ "...제기랄"(시간기반) → 암전 → 비명 → 타이틀
```

#### 4-2. 맵 구조
`TUT_MAP_W=18, TUT_MAP_H=9`. 시작(3,4)에서 통로 4칸 뒤 병원체 A/B 인접배치(danger=2 오렌지 시연, 통로 전이구간에 danger=1 옐로우 자연 발생). 그 뒤 더 연장된 통로 끝에 병원체 C 단독배치(좀비 습격 트리거). C 아래 3칸 통로에서 좀비가 화면을 가로질러 달려옴.

#### 4-3. 치료제 학습 — 2종류 명확히 구분
1. **평시 자율선택**(2번째 회수 직후): "치료제를 사용하시겠습니까?" Y/N, Y면 D키로 실제 사용 — **D키 사용법을 처음 가르치는 자리**.
2. **전투 중 강제선택**(80% 게이지, 식별 안 된 위협): 본게임 그대로의 Y/N 메커니즘, 단 튜토리얼은 결정시간 7초(본게임 3초보다 여유).
두 선택 결과가 직접 연결됨 — 평시에 치료제를 미리 썼으면 전투 시작 대사가 달라짐("제길, 치료제가...!" vs "치료제를 사용해야 하나? 식별이 안 됐는데!!").

#### 4-4. 전투 난이도 — F 10번 안팎으로 튜닝
드레인은 본게임 그대로(위기감 유지), `playerPower`만 18→11로 낮춰 연타횟수 조정(시뮬레이션 검증: 9~12회 수렴). `mashTimer`는 2.2배(8.8초)로 늘려 텍스트 읽을 여유 확보. 70%강제안내/게이지바 진행률 계산 모두 늘어난 총량 기준으로 재계산(원본 4초 기준으로 계산하면 비율 깨짐 — 실제 겪은 버그).

#### 4-5. 비네팅 시스템 신설
`TUT_VIGNETTE`: `mild`(회수 중 약함)/`aftermath`(전투후 강함, 펄스)/`precise_release`(정밀소나 핑 확산 비율에 직접 매핑해 해제). 미니게임보다 먼저 그려야 함(카메라변환 일시해제 후 화면좌표로).

#### 4-6. CONFIG 격리
본게임에서 산 특성(업그레이드)이 튜토리얼에 영향 주지 않도록 `resetConfigToDefaults()` 신설. 미니게임 패턴 길이도 튜토리얼은 항상 고정값.

#### 4-7. 산소/감염
대화창을 보는 동안(`TUT_LOCKED`)만 자연감소 정지, 그 외(자유이동/전투/연출)는 정상 감소 — 시행착오 끝에 이 기준으로 정착.

#### 4-8. 텍스트 엔진 안정화
타이핑은 텍스트노드+커서 구조(커서 분리로 인한 정렬 버그 해결). 타이핑 중 즉시완성(Space 스킵) 기능은 제거(사용자 요청 — 템포 유지), 완료 후 Space로 다음 진행하는 기능만 유지. `'timer'` 모드 연속 호출 시 뒤늦은 정리 타이머가 새 텍스트를 지워버리는 버그를 토큰 시스템(`_tutBoxToken`)으로 해결.

> **상세 구현 이력(시행착오 포함)은 `/mnt/user-data/outputs/OUTBREAK_ZONE_튜토리얼_정리.md` 참조** — 이번 갱신 시점까지의 모든 버그/수정 히스토리가 기록되어 있음.

## 5. ⚠️ ORIGIN 암전 연출 — 오해 주의 (절대 건드리지 말 것)
- 크리쳐 워프 시 `triggerOriginFlash` **항상 100% 발동**. 암전→화이트아웃 0.55초.
- **이건 의도된 것.** "물리적으로 이동했다"를 보여주는 목적. 과거엔 ORIGIN 비웃음 때문에 확률이었으나 폐기. **확률로 되돌리지 말 것.**
- **튜토리얼 습격 전투의 N선택/시간초과 승리는 예외** — `warpZombie()`(워프) 대신 즉시 `zombies.splice`로 제거. 좁은 튜토리얼 맵엔 워프 후보지(거리 9칸 이상 FLOOR)가 없어 스턴만 걸리고 좀비가 다시 인식하던 버그 때문. **본게임 로직은 그대로 유지, 튜토리얼만 분기.**

## 6. 🔴 미해결 / 진단 중

### 6-1. [신규·우선] 튜토리얼 신규 입력의 모바일 터치 대응 미검증
출품 타깃이 모바일인데, 튜토리얼에서 새로 만든 입력(평시 치료제 Y/N, D키 안내, G키 정밀소나)이 `_updateTouchUI()`/터치 버튼(`#touch-y`,`#touch-n` 등)과 실제로 연결되는지 코드상 확인 결과 **연결 안 됨**. `_updateTouchUI`가 `minigame.serumChoice` 기준으로 버튼을 노출하는데, 평시 치료제 선택은 `TUT_STEP` 기준이라 다른 트리거 체계임. 실기기/터치 에뮬레이션 확인 및 보강 필요.

### 6-2. [기존 이슈, 미해결] 모바일 타일 크기 (PC보다 큼 = 시야 좁음)
- 동적 ZOOM + orientationchange 추가했으나 **여전히 해결 확인 안 됨**.
- game.js **~1345줄 `[RESIZE]` 디버그 콘솔 로그 아직 남아있음**(mobile/W_px/H_px/ZOOM/innerWidth/innerHeight). 모바일에서 값 확인 후 원인 확정 + 로그 제거 필요.
- 의심: 가로 회전 시 innerWidth가 세로값 반환 또는 갱신 지연. devicePixelRatio 미사용(논리픽셀).

### 6-3. 캐릭터 크기
- 모바일에서 약간 작아보임. 타일/줌 이슈 해결 후 점검.

### 6-4. [신규] 튜토리얼 습격 후 setTimeout 가드 부재
`onTutorialAmbushResolved`/`onTutorialPreciseFired`의 중첩 setTimeout(최대 3.3초 지연) 콜백에 `TUT_ACTIVE`/`TUT_STEP` 가드 없음. 극단적 케이스(그 사이 강제 상태변경)에서 예약된 콜백이 오작동할 수 있음. 우선순위는 낮음(일반 플레이에선 발생 안 함).

### 6-5. 사운드 확인 필요
`gameover_infected`를 튜토리얼 최종 암전 직전 "비명"으로 재사용 중 — 실제로 비명처럼 들리는지 직접 청취 필요.

## 7. 임시 코드 (제거/재검토 표식)
- **`ensureUnit00Fallen()`** (~2613/5192줄): UNIT-00 강제 전사자 등록. **튜토리얼이 이제 완성됐으므로 이 함수의 필요성을 재검토할 시점.** 다만 "튜토리얼 재플레이 없이 바로 본게임 테스트"용으로 의도적으로 남겨둔 것이라, 제거 여부는 사용자 판단 필요 — 단순 임시코드가 아니게 됨.
- `lang = 'ko'` (~364줄대): 다국어 하드코딩. 'en'으로 바꾸면 영어 전환. VOICE/pickVoice.
- **`[RESIZE]` 로그** (~1345줄): 모바일 줌 진단용. 6-2 해결 후 제거.
- **`startTutorial()` 상태 초기화 누락**: `TUT_FREEZE_ZOMBIES`, `TUT_VIGNETTE.active`, `TUT_FORCE_MASH_SHOWN`, `TUT_SERUM_PROMPT_TIMER`가 `startTutorial()` 시작부에서 명시적으로 리셋되지 않음. 정상 완주 후엔 일부 정리되나, 콘솔로 강제 초기화 후 재실행하는 워크플로우에서 이전 잔여값이 남을 수 있음.
- **병원체 개수 HUD 하드코딩**: `updateHUD()`의 `const total = TUT_ACTIVE ? 3 : ...`가 `MAP.mineCount`와 별개로 숫자가 박혀 있음(맵을 2→3개로 늘렸을 때 한 번 이 값을 깜빡해 "1/2" 버그가 났던 자리). `MAP.mineCount` 참조로 교체하면 향후 맵 변경 시 자동 동기화됨.
- **타이핑 엔진 3중 복제**: 엔딩 터미널/풀스크린 인트로/하단 대화창에 거의 동일한 타이핑 엔진이 복제됨. 같은 버그(중복출력, 즉시완성)를 두 곳에 각각 적용해야 했던 전례 있음 — 공용 함수로 추출 권장.
